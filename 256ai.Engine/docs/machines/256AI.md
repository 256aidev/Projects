| Field | Value |

**Why 256AI is Coordinator:**
- Always on (stable)
- MainWin is NOT always available
- Already runs many services → light additional load acceptable

**Coordinator Responsibilities:**
- Receive tasks from Human (Strategy Layer)
- Decompose into subtasks
- Dispatch to workers (AI2, Mac, MainWin)
- Approve worker plans before execution
- Process escalations, provide dispositions
- Assemble final results
- Report to Human

## Ollama Access (For Apps)

Apps should call Ollama at:
- **Local (on 256AI):** `http://localhost:11434`
- **Network (from other machines):** `http://10.0.1.147:11434`

**Network config (if needed):**
```powershell
# PowerShell (run as admin)
[System.Environment]::SetEnvironmentVariable("OLLAMA_HOST", "0.0.0.0:11434", "Machine")

# Restart Ollama
taskkill /IM ollama.exe /F
ollama serve
```

Also add Windows Firewall rule for port 11434 inbound.
